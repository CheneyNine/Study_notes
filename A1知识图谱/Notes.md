 Software Development Kit Design

## Notes

SDK 就是 Software Development Kit 的缩写，中文意思就是【软件开发工具包】。这是一个覆盖面相当广泛的名词，可以这么说：辅助开发某一类软件的相关文档、范例和工具的集合都可以叫做【SDK】。

SDK 的使用者主要是 B 端客户，最终交付产品是代码、示例和文档，客户接入 SDK 也是和 SDK 提供方交流的过程，对外沟通的成本比对内更高，遇到的问题也会更多。所以 SDK 开发对开发者的要求比对应用开发更高。能开发好 SDK 一定能开发好应用，但能开发好应用，未必能开发好 SDK。
链接：https://www.zhihu.com/question/286904345/answer/2136791121

理想情况：开发者只需要关心调用哪些接口、传递哪些参数，就跟调用自己写的代码一样简单.

API也是可以调用第三方的程序，SDK也可以使用第三方的软件。这两者有什么区别吗？

SDK相当于是一个开发者集成的环境，API则是数据接口，API是基于SDK之下的，可以在SDK的环境之下调用API数据。

SDK包括了API的定义，API是定义了一种能力的属性，是一种接口的规范，并且，SDK也包含了API的能力和规范，还有一些其他的辅助功能，同是缺少一部分API的能力。



一个基本的 SDK 通常由编译器、调试器和应用编程接口（API）组成，但也可能包含以下任意内容：

- 文档
- 库
- 编辑器
- 运行时/开发环境
- 测试/分析工具
- 驱动程序
- 网络协议



**二、SDK开发的基本流程**

 

1. 确定需求：首先，您需要确定您的应用程序需要什么样的功能和特性，比如音频、视频、网络连接等。

 

2. 选择SDK：根据您的需求，选择适合您的SDK，比如我们今天要介绍的手机SIP软电话SDK开发包。

 

3. 下载SDK：下载您选择的SDK，并将其集成到您的开发环境中。

 

4. 阅读文档：仔细阅读SDK的文档，了解SDK的使用方法和注意事项。

 

5. 编写代码：根据SDK的文档和示例代码，编写您的应用程序代码。

 

6. 调试测试：进行代码调试和测试，确保您的应用程序能够正常运行。

 

7. 发布上线：最后，将您的应用程序发布到应用商店或其他渠道上线。



**三、SDK开发的技巧**

 

1. 熟悉SDK的文档：仔细阅读SDK的文档，尤其是API文档，可以更好地了解SDK的使用方法和参数，从而更有效地开发应用程序。 

2. 使用示例代码：SDK通常会提供示例代码，可以用于快速了解SDK的使用方法和特性。可以基于示例代码进行二次开发，提高开发效率。 

3. 注意安全性：在使用SDK时，需要注意安全性，比如防止信息泄露、防止恶意攻击等。 

4. 保持更新：SDK的版本会不断更新，需要及时了解和更新SDK，以获得更好的使用体验和更多的功能。
